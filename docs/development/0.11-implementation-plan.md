# 0.11 stable contracts and security implementation

This repository-grounded plan records how the 0.11 gate was delivered without
refactoring the runtime state machine.

## Phase 1 — REST contract

- Replace public `ProcessDefinitionEntity`, `ActivityHistoryEntity`, and
  mutation maps with DTOs in `com.abada.engine.dto`.
- Keep `/api/v1`, array-shaped list bodies, existing JSON names and mutation
  status behavior; add bounded database pagination and filters.
- Standardize failures through `GlobalExceptionHandler`, `ApiErrorController`,
  `ApiErrorCode`, and `ErrorResponse`.
- Validate generated SpringDoc output against
  `contracts/api-v1-contract.json` in `OpenApiContractTest` and CI.

## Phase 2 — Worker protocol

- Extend `ExternalTaskController`, `ExternalTaskCommandService`, worker DTOs,
  `ExternalTaskEntity`, and `ExternalTaskRepository` for protocol v1.
- Add ownership/expiry validation, bounded batches, heartbeat, BPMN error and
  trace metadata while retaining `/extend-lock` and local/test legacy
  completion compatibility.
- Add Flyway V9 and V1–V8 upgrade evidence.

## Phase 3 — Security

- Replace coarse path authorization in `SecurityConfig` with deployment,
  process, task, operations and worker domains.
- Map OIDC/proxy identity without treating BPMN groups as permissions.
- Return typed 401/403 bodies and prove invalid/expired JWT, forged proxy
  headers, role boundaries, CORS and logging behavior.

## Phase 4 — Products and SDK

- Add the independently buildable Java 21 module under `sdk/java`.
- Correct Tenda's process/definition/error types.
- Correct Orun incident mapping and use durable activity history instead of
  synthesizing audit data from active tokens.

No runtime-wide abstraction, new broker, or repository module was introduced.
V9 is additive and non-destructive.
